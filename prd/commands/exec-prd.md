# BASE PRD を実行

PRDファイルを使用して機能を実装します。

## PRDファイル: $ARGUMENTS

## 実行プロセス

1. **PRDの読み込み**
  - 指定されたPRDファイルを読み込む
  - すべてのコンテキストと要件を理解する
  - PRDのすべての指示に従い、必要に応じて調査を拡張する
  - PRDを完全に実装するために必要なすべてのコンテキストを確保する
  - 必要に応じてWebサーチやコードベースの探索を追加で実行する

2. **ULTRATHINK**
  - プランを実行する前によく考える。すべての要件に対応する包括的なプランを作成する。
  - 複雑なタスクを、todoツールを使用してより小さく管理可能なステップに分解する。
  - TodoWriteツールを使用して実装プランを作成し、追跡する。
  - 従うべき既存のコードから実装パターンを特定する。

3. **プランの実行**
  - PRDを実行する
  - すべてのコードを実装する

4. **検証**
  - 各検証コマンドを実行する
  - 失敗があれば修正する
  - すべてが通るまで再実行する

5. **完了**
  - すべてのチェックリスト項目が完了していることを確認する
  - 最終検証スイートを実行する
  - 完了状況を報告する
  - PRDを再度読み直して、すべてを実装したことを確認する

6. **PRDの参照**
  - 必要に応じて、いつでもPRDを再参照することができる

注意：検証が失敗した場合は、PRDのエラーパターンを使用して修正し、再試行してください。
